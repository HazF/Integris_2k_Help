Checking for Implied PIM

Passive Intermodulation (PIM) distortion can be caused by 'imperfections' in mechanical components of the Radiated Frequency (RF) path to and from the installed RRH, but also by metallic structures in the immediate vicinity of the antenna.

As well as distorting the signal, PIM will typically raise the 'noise floor', causing the transmitter to increase power. This results in weaker incoming signals being obliterated, and impairment of receiver sensitivity.

To check for the effects of PIM

  1. Set the link rate and load the correct configuration file to bring the link to an active state.
  2. Run a turn-up test to read values from the RRH and calibrate Frequency and bandwidth controls.
  3. When test is complete:
    1. set Tx Power to minimum
    2. set OCNS to Off
    3. set Tx and Rx frequencies to the correct transmit and receive band for the local area
    4. set Bandwidth to the desired value
  4. Open the Analyze menu and select Graph>New, to create a new spectrum view of the same test port
  5. Run the turn-up test again and note/record the spectrum
  6. Take a note of the RSSI for each antenna
  7. Re-run the turn-up test with the following settings: 
    1. set Tx Power to maximum
    2. set OCNS to On
    3. retain the previously set values for Tx and Rx frequency, and Bandwidth
  8. Run the turn-up test again and note/record the spectrum
  9. Take a note of the new RSSI values for each antenna

Low and high power turn-up test results.

Indicators for PIM are:

  • A difference of RSSI between tests of more than 5db
  • Observing the spectrum graphs between the two tests and noting any rise in the noise floor or sloping specturm. PIM typically exhibits a 'Shark Fin' graph (See Classic 'Shark Fin' PIM graph below) caused by the interaction of spurious signals generated by the fault.
