Installation Test Sequence

The Integris 2000 Install test checks the operation of the whole RAN installation and successful completion of this test signals a Pass for the site. This section describes a complete RAN installation test sequence, then records the results to a report file. This file can be exported in Portable Document Format (PDF) for record keeping purposes.

Radio Head testing consists of a sequence of individual tests that initialize the CPRI link and Remote Radio Head (RRH). They then verify that the link is capable of reliably supporting CPRI traffic at the current speed.

 

Connecting Integris 2000 to an RRH

Step 1: Connecting to the RRH To test or monitor a radio head installation connect the cable assembly to an Integris test port.

  1. Remove cable connecting the RRH to the BBU, by disconnecting it from the SFP in the BBU
  2. Remove the SFP from the BBU port where the cable to the RRH was previously connected
  3. Place this SFP in the desired Integris test port

  4. Connect the cable from the RRH to the SFP in the Integris test port 1 (See Fig 1.2 ).

  5. Power on the Radio Head.
 

Integris 2000 Test Port 1

Step 2: Setting the link rate:

  1. Click on the Rate icon in the Status Panel at the top left of the main window. A drop down menu will appear. The currently selected rate is highlighted
  2. Click on the required link rate. (See Fig 1.3 )
 

Link rate setting

The text on the icon will display the words "Active" in green if the SFP recognizes an active CPRI link, or a red error message if a problem exists.

Step 3: Loading a configuration file:

Radio Head testing is accomplished by transmitting a test sequence that is matched to the equipment being tested. These tests and pass/fail criteria are defined by a configuration file, which must be loaded before any testing can take place.

  1. Click on the Test Config icon at top left of the main screen

  2. Select the required configuration file from the list.

    (See See "Configuration Files", page 1 for more details on configuration files)

    (See Configuration Files for more details on configuration files)

Step 4: Creating a test report:

Report files contain the results of individual component tests. The first step in a test sequence therefore is to create a report file to contain them. The report file will contain one or more sets of results from the tests you run.

When you start a test a default report is created for you, but it is a good idea to create one ahead of testing when you can edit the name and add extra details such as location.

  1. Select the Report menu and click New. A new blank report will be created with a default format file name and the current date.

    Alternatively, click on Select to open a previously saved report.

  2. On the Report menu, click on Edit. In the report window that opens add site details and any notes you might want to include. You can also edit the file name to something more appropriate.

 

Editing report details

Note: You can go back and edit report fields at any time by clicking on Report > Edit.

Now that the report has been created and any required details added you can run the installation test sequence.

Step 5: Running the Test:

  1. On the Test menu, click Select Test, then Install.
  2. Start the test by clicking on Test > Run (or pressing F5)
  3. When the test has completed a yellow message box will open offering to save the results to the currently open report file. Click on OK

Step 6: Processing the report file

To see the report containing the results of individual tests:

  1. Go to the Report menu and click View (or click Ctrl + R).
  2. The screen will now display the report containing a list of individual test results that can be expanded or collapsed using the arrows to the left of the result name.
     

    Report showing test results

Step 7: Exporting the report file in Portable Document Format (PDF)

  1. Click on Report > Export PDF. An Explorer window will open allowing you to name and save the PDF file to a location of your choice.

  2. Click SAVE.
